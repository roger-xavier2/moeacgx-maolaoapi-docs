Images API

异步图片任务

提交 OpenAI Images 兼容请求，立即获取任务 ID，然后轮询状态并读取图片。 适合生成时间较长、同步请求容易超时的场景。

POST https://maolaoapi.com/v1/images/tasks
i
提交成功不等于生成成功

HTTP 202 只表示任务已接收。参数校验、渠道选择、余额和上游错误会在任务状态中返回。

1

提交

发送标准生图请求，取得 task_id。

2

轮询

每 2～3 秒查询一次，直到成功或失败。

3

取图

使用结果 URL 下载图片，或先获取 Blob 再展示。

开始调用

所有请求都要携带 API Key

在请求头中放入 MaolaoAPI 令牌。示例中的密钥只是占位符，不要把真实密钥写进前端源码或公开仓库。 

Authorization: Bearer sk-你的API_KEY

任务归属于令牌对应的账号。同一账号下的其他有效令牌可以查询任务，其他账号会得到 404 task not found。

创建任务

提交图片生成请求

请求体与 /v1/images/generations 相同。不传 action 时，默认创建图片生成任务。 

curl -X POST "https://maolaoapi.com/v1/images/tasks" \
  -H "Authorization: Bearer sk-你的API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "一只坐在月球上的橘猫，电影级光影",
    "size": "1024x1024",
    "quality": "high",
    "n": 1,
    "response_format": "b64_json"
  }'

成功接收

服务器返回 HTTP 202 Accepted：

{
  "task_id": "task_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "status": "queued"
}
!
当前响应不包含 request_id

保存 task_id，后续查询和读取图片都使用这个值。

常用参数

model图片模型名称。建议始终显式填写。
prompt图片描述或编辑指令。
n请求图片数量，默认 1。网关上限 128，渠道可能部分交付。
size例如 1024x1024，也可由渠道映射为宽高比。
quality例如 auto、standard、high 或 2K。
response_format建议使用 b64_json，结果会转换成轻量图片地址。

支持的 action

  • 不传、generations、images/generations：图片生成。
  • edits、images/edits：图片编辑。
轮询任务

使用 task_id 查询状态

GET /v1/images/tasks/{task_id} 
curl "https://maolaoapi.com/v1/images/tasks/task_xxx" \
  -H "Authorization: Bearer sk-你的API_KEY"

处理中

{
  "task_id": "task_xxx",
  "status": "processing",
  "progress": "10%"
}

生成成功

{
  "task_id": "task_xxx",
  "status": "succeeded",
  "progress": "100%",
  "expires_at": 1784200000,
  "result": {
    "created": 1784196400,
    "data": [
      {
        "url": "/v1/images/tasks/task_xxx/content/0",
        "revised_prompt": "..."
      }
    ]
  }
}
!
以实际返回数量判断交付结果

succeeded 表示至少交付了一张图片，不保证 result.data.length 与请求的 n 相等。请求多张图片时，请始终以 result.data.length 作为实际交付数量。

固定按张计费时按实际可交付数量结算，且不超过请求的 n；没有可交付图片时会进入失败、重试或退款流程。

生成失败

{
  "task_id": "task_xxx",
  "status": "failed",
  "progress": "100%",
  "error": "上游返回的错误原因"
}
queued任务已接收
processing正在调用上游
succeeded至少交付一张图片
failed任务执行失败

progress 是阶段性展示值，不是上游实时进度。业务判断应以 status 为准。

获取结果

下载受保护的图片内容

如果 result.data[].url 以 /v1/ 开头，请拼接 API 域名，并继续携带 Bearer Token。 当上游结果包含 Base64 图片内容时，网关会优先返回这种稳定的本站内容地址。 

curl "https://maolaoapi.com/v1/images/tasks/task_xxx/content/0" \
  -H "Authorization: Bearer sk-你的API_KEY" \
  -o result.png

内容接口直接返回图片二进制，支持 PNG、JPEG、WebP、GIF 和 AVIF。如果上游只返回 URL，结果可能是完整的 https:// 地址，此时直接使用该上游地址。

i
浏览器需要先获取 Blob

普通 <img src> 不能附加 Authorization 请求头，不要直接使用受保护的相对地址。 

const response = await fetch(
  `https://maolaoapi.com/v1/images/tasks/${taskId}/content/0`,
  {
    headers: { Authorization: `Bearer ${apiKey}` },
  },
)

if (!response.ok) {
  throw new Error(`图片读取失败：${response.status}`)
}

const blob = await response.blob()
const imageURL = URL.createObjectURL(blob)
document.querySelector('#result-image').src = imageURL
图片编辑

使用 multipart/form-data 上传原图

把 action=edits 放在查询参数中。mask 可选，最终能力取决于模型和渠道。 

curl -X POST "https://maolaoapi.com/v1/images/tasks?action=edits" \
  -H "Authorization: Bearer sk-你的API_KEY" \
  -F "model=gpt-image-2" \
  -F "prompt=把背景替换成夜晚的东京街道" \
  -F "image=@input.png" \
  -F "mask=@mask.png" \
  -F "size=1024x1024" \
  -F "quality=high" \
  -F "n=1"

异步执行只能避免等待生成结果时的 HTTP 超时，不能绕过 Cloudflare 或 Nginx 的请求体大小限制。

完整示例

提交任务并等待生成完成

const baseURL = 'https://maolaoapi.com'
const apiKey = 'sk-你的API_KEY'

async function createImageTask() {
  const response = await fetch(`${baseURL}/v1/images/tasks`, {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${apiKey}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      model: 'gpt-image-2',
      prompt: '一座漂浮在云海上的未来城市',
      size: '1024x1024',
      quality: 'high',
      n: 1,
      response_format: 'b64_json',
    }),
  })

  if (!response.ok) throw new Error(await response.text())
  return response.json()
}

async function waitForTask(taskId) {
  while (true) {
    const response = await fetch(
      `${baseURL}/v1/images/tasks/${encodeURIComponent(taskId)}`,
      { headers: { Authorization: `Bearer ${apiKey}` } },
    )

    if (!response.ok) throw new Error(await response.text())
    const task = await response.json()

    if (task.status === 'succeeded') return task.result
    if (task.status === 'failed') {
      throw new Error(task.error || '图片生成失败')
    }

    await new Promise((resolve) => setTimeout(resolve, 2500))
  }
}

const submitted = await createImageTask()
const result = await waitForTask(submitted.task_id)

console.log(result.data)
渠道兼容

Imagen 可以使用同一条异步路径

请求体仍然采用 OpenAI Images 格式。当前 Gemini 图片适配器只支持最终上游名称以 imagen 开头的模型。 

curl -X POST "https://maolaoapi.com/v1/images/tasks" \
  -H "Authorization: Bearer sk-你的API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "imagen-4.0-generate-001",
    "prompt": "赛博朋克风格的重庆夜景",
    "n": 1,
    "size": "16:9",
    "quality": "2K",
    "response_format": "b64_json"
  }'
✓
支持

imagen-* 以及映射后上游为 imagen-* 的业务别名。

×
暂不支持

Gemini 原生 contents、parts、generationConfig 请求体。

Gemini 原生聊天生图模型仍应使用 /v1beta/models/{model}:generateContent 或已有的兼容聊天路径。

异常处理

同时处理 HTTP 状态和任务状态

200任务查询或图片读取成功。
202任务已接收，需要继续轮询。
400action、参数或当前任务状态不合法。
401API Key 缺失或无效。
404任务、图片序号不存在，或任务不属于当前账号。
410图片正文已经过期。
413请求超过反向代理或 Cloudflare 上传限制。 
{
  "error": {
    "message": "task not found",
    "type": "invalid_request_error"
  }
}
数据生命周期

图片正文默认保留 1 小时

管理员可以在性能设置中调整为 0～8760 小时。设置为 0 会关闭自动清理。到期后任务状态和审计记录仍然保留，但图片正文无法继续读取。 

{
  "task_id": "task_xxx",
  "status": "succeeded",
  "progress": "100%",
  "result_expired": true,
  "expires_at": 1784200000
}

当前限制

  • 暂不提供取消任务接口。
  • 暂不提供完成回调或 Webhook。
  • 暂不提供公开的任务列表接口。
  • ?group=xxx 不能覆盖 API Key 对应的分组。
  • 渠道选择和限流规则与同步图片接口一致。
  • 固定按张计费以实际可交付数量结算，最多不超过请求的 n。
